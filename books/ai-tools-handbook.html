AI-tools: Пайплайн с Claude и Cursor

Практический справочник по Claude, Cursor и Cowork. Не теория — конкретные команды, готовые промпты и правила распределения задач между инструментами.

Часть 1

Мой стек и модели Claude

Тарифы: когда что выбирать

  • Pro (~$17/мес) — среднее использование, ~100–200 запросов/нед. Если часто упираешься в лимиты — это сигнал переходить.
  • Max (~$100/мес) — power-user, 5–20x больше лимитов, включает Claude Code и Computer Use.
📌 Про токены и русский язык

Русский текст занимает в 2–3 раза больше токенов чем английский. Для промптов это не критично, но при анализе больших документов — учитывай. Контекстное окно Sonnet 4.6 — 1M токенов, это ~700 000 слов на русском.

Часть 2

Cursor: Auto vs Composer

Два разных инструмента

ИнструментКак вызватьЧто делаетЛимиты
Auto (Tab) Автоматически при наборе Дополняет строку или функцию, работает в фоне Минимальные, включено всегда
Composer Ctrl+K (Cmd+K на Mac) Анализирует весь проект, multi-file редактирование 3–10 запросов/операция, 1–2 раза/час
📌 Важно

Auto и Composer — это НЕ одно и то же. "Auto" в настройках Cursor не включает Composer. Выбор модели "Auto" означает автовыбор между Sonnet и GPT-4o — Composer надо вызывать отдельно через Ctrl+K.

Workflow без упора в лимиты

09:00–11:00  Auto (Tab) → пишешь новые фичи  (~20–30 Tab)
11:00        Composer   → "Review morning work"  (1 запрос)
13:00–15:00  Auto (Tab) → допиливаешь            (~20 Tab)
15:00        Composer   → "Final refactoring"    (1 запрос)
17:00        Composer   → "Prep for commit"      (1 запрос)

Итого: ~60 Auto + 3 Composer = хорошо вписывается в Pro $20

Часть 3

Claude Code

Что это

Claude Code — это Claude в терминале. Не плагин, не IDE. Запускается командой claude в командной строке и работает с файлами напрямую.

  • Видит весь репозиторий (1M+ токенов контекста)
  • Выполняет команды в терминале сам
  • Работает с Git: читает историю, делает коммиты
  • Приватность: обработка локальная, не в облаке
💡 Где взять

Claude Code входит в тариф Max. Устанавливается через npm: npm install -g @anthropic-ai/claude-code

Когда использовать Claude Code, а не Cursor

Часть 4

Cowork

Что это

Cowork — десктопное приложение Anthropic (бета). Claude с прямым доступом к файлам на твоём компьютере. В отличие от claude.ai — видит папки, читает и пишет файлы, работает как агент.

Coworkclaude.ai
Читает файлы с компьютера напрямуюТолько загруженные вручную
Пишет и редактирует файлыТолько показывает код
Работает как агент (несколько шагов)Один ответ за раз
Нет памяти между сессиямиНет памяти между сессиями

Реальные кейсы

Читает папку с MD-файлами, добавляет YAML-шапки, раскладывает по категориям
📊 Финансовый анализ
Принимает скриншоты банковских выгрузок, категоризирует транзакции, заполняет Google Sheets через Apps Script

Читает сырые конспекты, убирает галлюцинации GPT, оставляет только реально применимые вещи
🔧 Скрипты и автоматизация
Пишет и запускает Python/bash скрипты прямо на твоём компьютере, проверяет результат

Малоизвестные фишки Cowork

1. _context.md — мгновенный старт

Cowork не помнит предыдущие сессии. Положи файл _context.md в рабочую папку — в начале каждой сессии пиши прочитай _context.md. Он загрузит весь контекст за один запрос вместо 10 минут объяснений.

2. Батч-обработка экономит токены

Вместо "обработай файл 1, потом файл 2" — давай одну задачу на всю папку. Cowork сам разберётся с итерацией, а ты платишь за один системный промпт.

3. Отчёт вместо диалога

Добавляй в конец любого запроса: "не объясняй что делаешь — сделай и выдай отчёт в конце". Экономит 30–50% токенов на длинных задачах.

4. Apps Script как мост к Google

Cowork не может писать напрямую в Google Sheets (MCP работает только с Docs). Но он пишет Apps Script который ты вставляешь в таблицу один раз — и дальше данные пишутся через скрипт. Это стабильнее чем API.

5. Разные папки — разные контексты

Открывай Cowork из разных рабочих папок для разных задач. Он не будет путать базу знаний с финансами — контекст изолирован по папке.

📌 Про токены в Cowork

Первичная обработка большого количества файлов всегда дорогая — Cowork читает каждый файл полностью. Это разовая работа. После неё — работа с базой дешевеет в 3–5 раз потому что файлы маленькие и структурированные.

Правила экономии токенов

  • Читай только шапки — для метаданных файлов не нужен полный текст. Говори Cowork: "читай только первые 20 строк каждого файла"
  • Один большой запрос лучше пяти маленьких — контекст загружается один раз
  • Отчёт вместо диалога — "сделай и выдай отчёт" вместо пошагового обсуждения
  • Sonnet вместо Opus — для большинства задач Sonnet 4.6 справляется, Opus резервируй для сложного синтеза
  • _context.md — один файл вместо пересказа контекста каждый раз

Дорогая операция — та которую делаешь руками вместо скрипта. Токены дешевле твоего времени.
